---
title: "Quickstart"
description: "Get the Go Payment Dashboard running locally in under five minutes."
---

This guide takes you from zero to a running full-stack payment dashboard on your machine. You'll need about five minutes and Docker.

<Steps>
  <Step title="Install Docker">
    The entire stack runs via Docker Compose — it's the only local dependency you need.

    Download and install [Docker Desktop](https://www.docker.com/products/docker-desktop/) for your OS, then verify it's working:

    ```bash
    docker --version
    ```

    <Warning>
      Docker Desktop must be running before you start the stack. On Linux, you can use the Docker Engine directly without Docker Desktop.
    </Warning>
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/axelcarl/go-dashboard.git
    cd go-dashboard
    ```
  </Step>

  <Step title="Create the root .env file">
    Copy the example environment file and fill in your values:

    ```bash
    cp .env.example .env
    ```

    Open `.env` and set each variable. Here's a working example for local development:

    ```bash .env
    PORT=8080
    APP_ENV=development
    DB_HOST=db
    DB_PORT=5432
    DB_DATABASE=dashboard
    DB_USERNAME=postgres
    DB_PASSWORD=postgres
    DB_SCHEMA=public
    DB_SSLMODE=disable
    FRONTEND_ORIGIN=http://localhost:5173
    ```

    <Note>
      `DB_HOST` should match the Docker Compose service name for your database container if you're running Postgres inside Docker. `FRONTEND_ORIGIN` is used by the Go backend for CORS — it must match the address your browser uses to reach the frontend.
    </Note>
  </Step>

  <Step title="Create the frontend .env file">
    The React frontend needs to know where the Go API is running. Create `frontend/.env.local`:

    ```bash
    cp frontend/.env.production frontend/.env.local
    ```

    Then set `VITE_API_URL` to point at the backend:

    ```bash frontend/.env.local
    VITE_API_URL=http://localhost:8080
    ```

    <Tip>
      The port in `VITE_API_URL` must match the `PORT` value you set in the root `.env` file.
    </Tip>
  </Step>

  <Step title="Start the stack">
    From the project root, run:

    ```bash
    docker compose up
    ```

    Docker Compose will:
    1. Pull the `cosmtrek/air` image and mount your Go source files with live-reload enabled.
    2. Pull `node:25-alpine`, install frontend dependencies, and start the Vite dev server.

    The first run takes a minute or two while images download and `npm install` completes. Subsequent starts are faster.

    <Note>
      To run the stack in the background, use `docker compose up -d`. View logs at any time with `docker compose logs -f`.
    </Note>
  </Step>

  <Step title="Access the dashboard">
    Once both services are up, open your browser:

    <CardGroup cols={2}>
      <Card title="Frontend" icon="browser" href="http://localhost:5173">
        React dashboard at `http://localhost:5173`
      </Card>
      <Card title="API" icon="code" href="http://localhost:8080">
        Go REST API at `http://localhost:8080`
      </Card>
    </CardGroup>

    <Note>
      Replace `8080` with whatever `PORT` value you set in your `.env` file.
    </Note>
  </Step>

  <Step title="Verify the stack is healthy">
    Hit the health endpoints to confirm everything is connected:

    <CodeGroup>

    ```bash App health
    curl http://localhost:8080/health
    ```

    ```bash Database health
    curl http://localhost:8080/health/db
    ```

    </CodeGroup>

    A healthy response looks like:

    ```json
    { "status": "ok" }
    ```

    <Warning>
      If `/health/db` returns an error, your database connection details in `.env` are likely incorrect. Double-check `DB_HOST`, `DB_PORT`, `DB_USERNAME`, `DB_PASSWORD`, and `DB_DATABASE`.
    </Warning>
  </Step>
</Steps>

## What's running

| Service | Address | Description |
|---|---|---|
| Frontend | `http://localhost:5173` | React 19 + TypeScript dashboard (Vite dev server) |
| Backend API | `http://localhost:{PORT}` | Go REST API with chi router |
| Live reload | Automatic | Air watches Go files; Vite watches frontend files |
