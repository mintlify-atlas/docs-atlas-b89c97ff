---
title: 'Get payment'
description: 'Retrieve a single payment record by its ID.'
---

Returns a single payment object identified by its integer ID.

## Endpoint

```
GET /payments/{id}
```

## Path parameters

<ParamField path="id" type="number" required>
  The integer ID of the payment to retrieve.
</ParamField>

## Response

Returns the matching payment object.

<ResponseField name="id" type="number" required>
  Unique integer identifier for the payment.
</ResponseField>

<ResponseField name="sender" type="string" required>
  The name or identifier of the party sending the payment.
</ResponseField>

<ResponseField name="recipient" type="string" required>
  The name or identifier of the party receiving the payment.
</ResponseField>

<ResponseField name="amount" type="number" required>
  The payment amount as a floating-point number.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of when the payment was last updated.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the payment was created.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://go-dashboard.fly.dev/payments/42
```

```json Response
{
  "id": 42,
  "sender": "alice",
  "recipient": "bob",
  "amount": 250.00,
  "updatedAt": "2026-03-15T09:45:00Z",
  "createdAt": "2026-03-15T09:45:00Z"
}
```

</CodeGroup>

## Errors

| Status | Condition |
| ------ | --------- |
| `400` | The `id` path parameter is not a valid integer |
| `404` | No payment exists with the given ID |
| `500` | Database query failed |

<CodeGroup>

```json 400 response
{
  "error": "invalid payment id"
}
```

```json 404 response
{
  "error": "payment not found"
}
```

```json 500 response
{
  "error": "failed to retrieve payment"
}
```

</CodeGroup>
