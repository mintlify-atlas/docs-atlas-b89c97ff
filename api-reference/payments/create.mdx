---
title: 'Create payment'
description: 'Create a new payment record.'
---

Creates a new payment record and returns the created object with its assigned ID and timestamps.

## Endpoint

```
POST /payments
```

## Request body

All fields are required.

<ParamField body="sender" type="string" required>
  The name or identifier of the party sending the payment. Must not be empty.
</ParamField>

<ParamField body="recipient" type="string" required>
  The name or identifier of the party receiving the payment. Must not be empty.
</ParamField>

<ParamField body="amount" type="number" required>
  The payment amount. Must be greater than `0`.
</ParamField>

## Response

Returns the created payment object, including the server-assigned `id`, `createdAt`, and `updatedAt` fields.

<ResponseField name="id" type="number" required>
  The server-assigned unique integer identifier for the new payment.
</ResponseField>

<ResponseField name="sender" type="string" required>
  The name or identifier of the party sending the payment.
</ResponseField>

<ResponseField name="recipient" type="string" required>
  The name or identifier of the party receiving the payment.
</ResponseField>

<ResponseField name="amount" type="number" required>
  The payment amount.
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of when the payment was created (equals `createdAt` on creation).
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the payment was created.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://go-dashboard.fly.dev/payments \
  --header 'Content-Type: application/json' \
  --data '{
    "sender": "alice",
    "recipient": "bob",
    "amount": 99.99
  }'
```

```json Response
{
  "id": 7,
  "sender": "alice",
  "recipient": "bob",
  "amount": 99.99,
  "updatedAt": "2026-03-15T11:20:00Z",
  "createdAt": "2026-03-15T11:20:00Z"
}
```

</CodeGroup>

## Errors

| Status | Condition |
| ------ | --------- |
| `400` | Request body is not valid JSON |
| `400` | `sender` is empty |
| `400` | `recipient` is empty |
| `400` | `amount` is `0` or negative |
| `500` | Database insert failed |

<CodeGroup>

```json 400 — empty sender
{
  "error": "sender can't be empty"
}
```

```json 400 — empty recipient
{
  "error": "recipient can't be empty"
}
```

```json 400 — invalid amount
{
  "error": "amount must be greater than 0"
}
```

```json 500 response
{
  "error": "failed to create payment"
}
```

</CodeGroup>

<Warning>
  The `amount` field must be strictly greater than `0`. Passing `0` or a negative value returns a `400` error.
</Warning>
